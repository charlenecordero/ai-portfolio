## üèóÔ∏è Root Directory

### 1. `server.js`
**Role:** Backend Entry Point & API Gateway
-   **Runtime:** Node.js (Express server).
-   **Key Responsibilities:**
    -   **Static Serving:** Serves the `./public` directory as the frontend application.
    -   **API Wrapper:** Acts as a secure proxy to external services, keeping API keys (like Gemini) server-side.
    -   **`/api/chat` Endpoint:** Handles communication with Google Gemini. It constructs the "System Prompt" (defining the Persona "Aura") and feeds the user's message to the model.
    -   **`/send-email` Endpoint:** Uses `nodemailer` to send contact form submissions via SMTP (Gmail), creating a "transmission" effect.
    -   **Security:** Sets `Cache-Control` headers and handles standard middleware (cors, body-parser).

### 2. `cloudbuild.yaml`
**Role:** CI/CD Pipeline Configuration
-   **Platform:** Google Cloud Build.
-   **Logic:** Defines the steps that execute whenever code is pushed to the `main` branch.
    1.  **Build:** Runs `docker build` to create a container image from the source.
    2.  **Push:** Uploads the image to Google Artifact Registry (`asia-southeast1`).
    3.  **Deploy:** Updates the Cloud Run service (`ai-portfolio`) with the new image, ensuring zero-downtime deployment.

### 3. `Dockerfile`
**Role:** Container Definition
-   **Base Image:** `node:18-slim` (Lightweight Linux + Node.js).
-   **Build Steps:**
    -   Copies `package.json` and installs *production* dependencies.
    -   Copies source code.
    -   Runs `npm run build` to generate the CSS.
    -   Exposes the start command `npm start`.

### 4. `package.json`
**Role:** Dependency & Script Manifest
-   **Scripts:**
    -   `start`: Launches `server.js`.
    -   `build`: Compiles Tailwind CSS key using the **Tailwind CLI** (`@tailwindcss/cli`).
-   **Dependencies:** Key libraries include `@google/generative-ai` (AI), `express` (Server), `nodemailer` (Email), and `tailwindcss` (Styling).

## üé® Public Directory (`public/`)

### 1. `index.html`
**Role:** Single Page Application (SPA) Shell
-   **Structure:** The main HTML skeleton. It does *not* reload. Content navigation is handled by **Alpine.js**.
-   **Key Sections:**
    -   `x-data="appData()"`: The root element initializing the Alpine.js state.
    -   **Header**: Responsive navigation and Theme Toggle.
    -   **Content Views**: Distinct `<section>` blocks (Dashboard, About, Projects, etc.) that toggle visibility based on `currentSection`.
    -   **Modals**: Hidden-by-default overlays for "Project Details" and "Blog Posts".

### 2. `script.js`
**Role:** Frontend Logic & State Management
-   **Framework:** Alpine.js.
-   **Core Object (`appData`):**
    -   **State:** Tracks `currentSection`, `isLightMode`, `chatOpen`, and chart instances.
    -   **Data Stores:** Contains the raw JSON arrays for `experience`, `projects`, and `blogs`.
    -   **Function `toggleTheme()`**: Switches between light/dark mode and persists preference to `localStorage`.
    -   **Function `sendChatMessage()`**:
        -   First checks a list of **Offline Fallbacks** (regex matching) for instant answers to common questions ("Who are you?", "Explain Design").
        -   If no match, sends an async request to `/api/chat` (Server -> Gemini).
    -   **Chart Initialization:** Direct integration with Chart.js to render the "Career Trajectory" (Line) and "Tech Stack" (Doughnut) charts.

### 3. `style.css`
**Role:** Compiled Output Styles
-   **Status:** **Generated File**. Do not edit directly.
-   **Content:** Contains the final set of utility classes generated by Tailwind based on the usage in `.html` and `.js` files.

### 4. `code_architecture.html`
**Role:** Documentation View
-   **Purpose:** A standalone page explaining the "System Architecture" (the page you might be reading a version of).
-   **Design:** Mirrors the main site's aesthetic but focuses purely on technical diagrams and stack explanations.

## üíÑ Styles Directory (`styles/`)

### 1. `input.css`
**Role:** CSS Source of Truth
-   **Tailwind Directives:** `@import "tailwindcss";`
-   **Custom Layer (`:root`):** Defines the **CSS Variables** for the application's unique theme:
    -   `--neon-mint` (#4AF2A1) & `--neon-pink` (#FF2E88): Core accent colors.
    -   `--void-moss` (#050505): The deep background color.
-   **Theme Classes:**
    -   `.light-mode`: A scope that overrides the root variables with light-themed equivalents (e.g., White background, Dark Green text).
-   **Custom Components:**
    -   `.alien-border`: Uses `clip-path` to create sci-fi angled corners.
    -   `.glass-panel`: Backdrop blur and semi-transparent backgrounds.
    -   `.scanlines`: CSS animation simulating CRT monitor effects.
